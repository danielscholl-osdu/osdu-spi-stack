# OSDU Record Lifecycle

Complete workflows for creating, querying, updating, and deleting OSDU records.

## Prerequisites

Before creating records, you need:
1. An active connection: `uv run osdu.py connect`
2. A legal tag (CIMPL ships with `osdu-demo-legaltag`)
3. Knowledge of your ACL groups (check with entitlements)

### Discover ACL Groups

```bash
uv run osdu.py call GET /api/entitlements/v2/groups
```

For CIMPL, the default groups are:
- Viewers: `data.default.viewers@osdu.group`
- Owners: `data.default.owners@osdu.group`

### Check Available Legal Tags

```bash
uv run osdu.py call GET /api/legal/v1/legaltags
```

CIMPL bootstrap creates `osdu-demo-legaltag`.

---

## Create a Legal Tag

```bash
uv run osdu.py call POST /api/legal/v1/legaltags -d '{
  "name": "my-test-legaltag",
  "description": "Test legal tag for development",
  "properties": {
    "countryOfOrigin": ["US"],
    "contractId": "TEST-001",
    "originator": "DevTeam",
    "securityClassification": "Public",
    "personalData": "No Personal Data",
    "exportClassification": "EAR99",
    "dataType": "Public Domain Data",
    "expirationDate": "2027-12-31"
  }
}'
```

**Required fields:**
- `name`: Unique within partition (use timestamps for uniqueness)
- `countryOfOrigin`: Array of ISO 2-letter country codes
- `contractId`: Contract identifier
- `originator`: 3-60 chars, alphanumeric + spaces/hyphens/periods
- `securityClassification`: `Public`, `Private`, or `Confidential`
- `personalData`: `No Personal Data` or `Personally Identifiable`
- `exportClassification`: `EAR99`, `No License Required`, etc.
- `dataType`: `Public Domain Data`, `First Party Data`, etc.
- `expirationDate`: Future date in `YYYY-MM-DD`

## Create a Record

### Minimal Record

```bash
uv run osdu.py call PUT /api/storage/v2/records -d '[{
  "kind": "osdu:wks:reference-data--UnitOfMeasure:1.0.0",
  "acl": {
    "viewers": ["data.default.viewers@osdu.group"],
    "owners": ["data.default.owners@osdu.group"]
  },
  "legal": {
    "legaltags": ["osdu-demo-legaltag"],
    "otherRelevantDataCountries": ["US"]
  },
  "data": {
    "Name": "Test Unit",
    "ID": "test-unit-001"
  }
}]'
```

### Record with System-Generated ID

Omit `id` field -- OSDU generates one based on the kind:
```
osdu:reference-data--UnitOfMeasure:<auto-generated-hash>
```

### Record with Custom ID

```bash
uv run osdu.py call PUT /api/storage/v2/records -d '[{
  "id": "osdu:reference-data--UnitOfMeasure:my-custom-id",
  "kind": "osdu:wks:reference-data--UnitOfMeasure:1.0.0",
  "acl": { ... },
  "legal": { ... },
  "data": { ... }
}]'
```

### Batch Create (multiple records)

Pass an array of record objects. Each can have a different kind:
```bash
uv run osdu.py call PUT /api/storage/v2/records -d '[
  {"kind": "...", "acl": {...}, "legal": {...}, "data": {...}},
  {"kind": "...", "acl": {...}, "legal": {...}, "data": {...}}
]'
```

**Response:**
```json
{
  "recordIds": ["osdu:reference-data--UnitOfMeasure:abc123"],
  "recordIdVersions": ["osdu:reference-data--UnitOfMeasure:abc123:1234567890"],
  "skippedRecordIds": []
}
```

## Read Records

### Get by ID
```bash
uv run osdu.py call GET /api/storage/v2/records/{RECORD_ID}
```

### Get Specific Version
```bash
uv run osdu.py call GET /api/storage/v2/records/{RECORD_ID}/{VERSION}
```

### List All Versions
```bash
uv run osdu.py call GET /api/storage/v2/records/versions/{RECORD_ID}
```

### Batch Fetch
```bash
uv run osdu.py call POST /api/storage/v2/query/records:batch \
  -d '{"records": ["id1", "id2", "id3"]}'
```

## Search Records

### Find by Kind
```bash
uv run osdu.py call POST /api/search/v2/query \
  -d '{"kind":"osdu:wks:reference-data--UnitOfMeasure:*","limit":20}'
```

### Find by Data Field
```bash
uv run osdu.py call POST /api/search/v2/query \
  -d '{"kind":"osdu:wks:reference-data--UnitOfMeasure:*","query":"data.Name:(\"Test Unit\")"}'
```

### Find by Record ID (via search)
```bash
uv run osdu.py call POST /api/search/v2/query \
  -d '{"kind":"*:*:*:*","query":"id:\"osdu:reference-data--UnitOfMeasure:my-custom-id\""}'
```

See [search-patterns.md](search-patterns.md) for advanced query syntax.

## Update a Record

Send the full record with the same `id` to the create/update endpoint:
```bash
uv run osdu.py call PUT /api/storage/v2/records -d '[{
  "id": "osdu:reference-data--UnitOfMeasure:my-custom-id",
  "kind": "osdu:wks:reference-data--UnitOfMeasure:1.0.0",
  "acl": {
    "viewers": ["data.default.viewers@osdu.group"],
    "owners": ["data.default.owners@osdu.group"]
  },
  "legal": {
    "legaltags": ["osdu-demo-legaltag"],
    "otherRelevantDataCountries": ["US"]
  },
  "data": {
    "Name": "Updated Unit Name",
    "ID": "test-unit-001",
    "Description": "Added a description"
  }
}]'
```

This creates a new version. The previous version is still accessible via the
version-specific GET endpoint.

---

## Delete Records

### Soft Delete
```bash
uv run osdu.py call DELETE /api/storage/v2/records/{RECORD_ID}
```
Marks the record as deleted. Can potentially be recovered.

### Permanent Purge
```bash
uv run osdu.py call DELETE /api/storage/v2/records/{RECORD_ID}:purge
```
Permanently removes the record and all versions. Cannot be undone.

## Complete Workflow Example

```bash
# 1. Connect
uv run osdu.py connect

# 2. Check legal tags
uv run osdu.py call GET /api/legal/v1/legaltags

# 3. Check entitlements
uv run osdu.py call GET /api/entitlements/v2/groups

# 4. Create a record
uv run osdu.py call PUT /api/storage/v2/records -d '[{
  "kind": "osdu:wks:reference-data--UnitOfMeasure:1.0.0",
  "acl": {
    "viewers": ["data.default.viewers@osdu.group"],
    "owners": ["data.default.owners@osdu.group"]
  },
  "legal": {
    "legaltags": ["osdu-demo-legaltag"],
    "otherRelevantDataCountries": ["US"]
  },
  "data": {"Name": "My Test Record", "ID": "lifecycle-test-001"}
}]'

# 5. Verify via storage (immediate)
uv run osdu.py call GET /api/storage/v2/records/{RETURNED_RECORD_ID}

# 6. Wait for indexing, then search
# (wait ~30-60 seconds)
uv run osdu.py call POST /api/search/v2/query \
  -d '{"kind":"osdu:wks:reference-data--UnitOfMeasure:*","query":"data.ID:(\"lifecycle-test-001\")"}'

# 7. Clean up
uv run osdu.py call DELETE /api/storage/v2/records/{RECORD_ID}

# 8. Disconnect
uv run osdu.py disconnect
```
